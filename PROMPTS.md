# Prompts

The current build ships **no LLM calls in the audit path**. The executive
summary on the results page is generated by `generateSummary()` in
`src/lib/audit-engine.ts` — pure TypeScript, no API, no key.

This document captures the prompts we would use **if** we enable AI-augmented
summaries behind a feature flag in week 2.

## Goals

- Use AI as a **finisher**, not the source of truth.
- Numbers, names, and recommendations come from the engine. AI only rewords.
- Failure mode: fall back to the template summary, never block the result.

## Provider preference

1. Anthropic (Claude 3.5 Sonnet) — primary.
2. OpenAI (gpt-4o-mini) — fallback.

## System prompt

```
You are a senior FinOps analyst writing executive summaries for an AI-tool
spend audit. Your audience is a CTO or founder. Tone: confident, concrete,
zero fluff, no emojis, no exclamation marks.

Hard rules:
- Never invent numbers. Use ONLY the numbers in the input JSON.
- Never recommend actions that aren't in the recommendations list.
- 90–120 words. One paragraph. No bullet points.
- Lead with the headline savings figure.
- Reference the single highest-impact recommendation by action name.
- End with a forward-looking sentence about what changes next.
```

## User message template

```
Input:
{
  "teamSize": <number>,
  "totalCurrentMonthly": <number>,
  "monthlySavings": <number>,
  "yearlySavings": <number>,
  "topRecommendations": [
    { "action": "...", "monthlySavings": <number>, "reasoning": "..." }
  ]
}

Write the summary.
```

## Output validation

Before showing AI output:
- Length 60–180 words.
- Must contain `$` and a comma-separated number matching `yearlySavings`.
- Must NOT contain any number not present in the input.
- If validation fails, return the template summary.
